## Project Overview

**全国ダンスマップ (Zenkoku Dance Map)** — A Japanese dance school and event discovery website. Users select a region/prefecture on a map of Japan to browse dance schools, filter by type and area, and search by keyword. A separate page provides an event calendar.

## Development Setup

This is a static site with no build system, no package manager, and no framework. All code is vanilla HTML, CSS, and JavaScript (ES6+).

**To run locally**, you must use a local HTTP server (the CSV fetch fails on `file://` due to CORS):
```bash
# Python
python3 -m http.server 8000

# VS Code
# Use the "Live Server" extension
```

There are no tests, linters, or CI pipelines.

## Architecture

### Pages
- `home.html` — Landing page with hero video and site introduction
- `index.html` — Main feature: dance school finder with SVG map, filters, and search
- `events.html` — Event calendar (FullCalendar + Google Sheets via Tabletop.js)
- `contact.html` — Blog integration (note.com) and contact info

### Core Files
- `js/script.js` — All search page logic (~600 lines): CSV loading, parsing, region filtering, pagination, rendering
- `js/events.js` — Event calendar logic: Google Sheets fetch, FullCalendar config, date filtering
- `css/styles.css` — Single stylesheet for entire site (~25KB)
- `data/data_2_consolidated_clean.csv` — Master dataset of ~4500 dance schools (13 columns: area, name, tel, address, hp, type, hours, closed_days, comment, other, map, source_association, website)
- `data/events.json` — Fallback event data
- `includes/header.html` — Shared header fragment (not currently included dynamically in all pages)

### Data Flow (Search Page)

1. User clicks a region on SVG map or region button → `handleRegionClick(regionId)`
2. Fetches `data/data_2_consolidated_clean.csv` (single consolidated file, loaded each time)
3. Custom `parseCSV()` parser converts to object array (handles quoted fields, escaped quotes)
4. Filters data by matching `regionId` against `parentRegionMap` (prefecture→region mapping)
5. Generates sub-category filter tabs from the filtered data's `type` and `area` fields
6. `updatePagination()` applies type/area/free-word filters → `renderList()` outputs card HTML

### Key Data Structures
- `regionalCSVMap` — Maps region/prefecture IDs to CSV paths (legacy, currently unused; all data comes from consolidated CSV)
- `regionNameMap` — Maps romaji IDs (`tokyo`) to Japanese names (`東京`)
- `parentRegionMap` — Maps each prefecture to its parent region (e.g., `tokyo → kanto`)
- `selectedTypes` / `selectedAreas` — `Set` objects tracking active filter selections
- `currentData` — Array of parsed school objects for the currently selected region
- `currentPage` — Pagination state (10 items per page)

### Design System
- Color palette: sage green `#6b9e84` (primary), soft orange `#e6a86c` (accent), warm brown `#4a3f35` (text), cream gradient background `#faf7f2 → #f0ead6`
- Glassmorphism effects via `backdrop-filter` and semi-transparent backgrounds
- Fonts: Inter + Noto Sans JP (body), Kiwi Maru (decorative)
- Responsive breakpoint at 768px (mobile/desktop)

### External Dependencies (CDN)
- FullCalendar — Calendar widget on events page
- Tabletop.js — Google Sheets reader for events
- Animate.css — Logo pulse animation
- Google Fonts — Noto Sans JP, Inter, BIZ UDGothic, Kiwi Maru

## Design Rules

### 禁止パターン
- グラデーション禁止（`background: linear-gradient` など）
- ハードコードカラー禁止（CSS変数を使うこと）

### UI方針
- UIテキストはすべて日本語
- 高齢者対応：フォントサイズは16px以上、タップ領域は44px以上、コントラスト比4.5:1以上
- 装飾より可読性を優先

## Important Notes

- The `unused/` directory contains legacy data scrapers, old CSV files, and backup assets — not part of the active site
- CSV column `ダンス教室` is mapped to `type` and `allow` is mapped to `other` during parsing (see `headerMap` in `parseCSV`)
- The `area` column in the CSV stores romaji IDs (e.g., `tokyo`) which are converted to Japanese during parsing
- Google Maps links are generated by encoding the address into a search URL
- Events are loaded from a public Google Spreadsheet via Tabletop.js
